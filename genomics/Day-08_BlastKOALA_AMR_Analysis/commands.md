# Commands Used

## Step 1 — Prepare Protein FASTA File

The predicted protein sequences generated by Prokka (`.faa`) are used as input for BlastKOALA.

---

## Step 2 — Open BlastKOALA

https://www.kegg.jp/blastkoala/

Purpose

Upload protein sequences for KEGG Orthology (KO) assignment.

---

## Step 3 — Upload Protein File

Input File

```text
sample.faa
```

Purpose

Submit predicted protein sequences for functional annotation.

---

## Step 4 — Select Database

Choose the appropriate KEGG GENES database according to the organism.

Purpose

Improves annotation accuracy.

---

## Step 5 — Submit Job

Wait for BlastKOALA to complete the analysis.

---

## Step 6 — Download Results

Expected files include:

- KO Assignment
- Functional Annotation
- KEGG Pathway Mapping

---

## Step 7 — AMR Analysis

Upload genome or protein sequences to the AMR analysis tool used during the practical (e.g., AMRFinderPlus or another database).

Expected Output

- Antimicrobial resistance genes
- Resistance classes
- Gene annotations
